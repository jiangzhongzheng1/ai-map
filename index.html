关键事实
AI没有在"思考"——它在对你发来的文字做一次巨大的数学运算，预测接下来该输出什么词。每次都是全新的运算，从零开始。
01
大模型 / LLM

所谓"AI"
本质是什么

LLM（Large Language Model，大型语言模型）是一个极其巨大的数学函数。给它输入一段文字，它输出下一段文字。

它"懂"语言、"会"推理、"有"知识，是因为训练时看过数万亿字的人类文字——书、网页、代码、对话——然后调整内部数十亿个数字参数，直到能预测"什么词应该接在什么词后面"。

颠覆认知
模型本质上是一个超复杂的"下一个词预测机器"。它没有意识，没有情感，没有目标。它只是在做一件事：给定输入，预测最合理的续写。只不过这件事被做到了极致。

训练一次大模型需要几个月时间、数千张GPU、数亿美元。训练完成后，所有的"知识"都被压缩进了那些数字参数里，形成一个大文件——就是"模型权重"。

// 极度简化的理解 function LLM(input_text) { // 内部有几百亿个数字参数 // 训练了几个月 // 看过数万亿字的人类文字 return most_likely_continuation(input_text); }

模型有名字——Claude Sonnet、GPT-4o、Gemini 1.5 Pro、LLaMA 3——不同的模型是不同公司训练的，能力、风格、价格都不同。这和"操作系统"的关系很像：iOS和Android都是手机系统，但内部完全不同。

02
Token

AI 怎么数钱的

AI不按字、不按词处理文字，而是按 Token——一种介于字符和单词之间的单位。英文大约每4个字符是1个Token；中文大约每1-2个字是1个Token。

Token是AI世界里的"计量单位"——所有费用、所有限制，都以Token来衡量。

03
API

程序员的电源插座

API（Application Programming Interface）是模型对外开放的标准化接口——你写代码，把消息发进去，回答出来。不需要自己训练模型，直接调用别人已经训练好的。

用API就像用电——你不需要建发电厂，只需要知道插座怎么用。发电厂（模型）是Anthropic/OpenAI建的，插座（API）他们开放给你，你按用电量（Token）付费。

一次最简单的API调用长这样：

// 向Claude发一条消息 fetch("https://api.anthropic.com/v1/messages", { method: "POST", headers: { "x-api-key": "你的密钥", "Content-Type": "application/json" }, body: JSON.stringify({ model: "claude-sonnet-4-5", max_tokens: 1024, messages: [{ role: "user", content: "今天天气怎么样？" }] }) })
API vs 网页产品 的区别
用网页版（claude.ai）：有界面、有记忆感、有工具——但你控制不了任何细节。

用API：裸模型，没有记忆、没有系统指令、没有工具——但你完全控制一切，可以自己加。这是开发者选API的原因。

API不只给大公司用。任何人申请一个API Key（一般几分钟），就可以在自己的程序里调用世界最强的AI。这是个人开发者 / 独立产品的基础设施。

04
上下文窗口

AI 没有记忆
你每次都在发历史

这是理解AI系统最重要的一件事，也是最多人不知道的。

AI没有持久记忆。它不记得上一条消息。每次请求对它来说都是全新的、独立的运算。

那为什么聊天软件里AI"记得"之前说了什么？因为每次发消息，系统都把完整的对话历史一起打包发给AI。

模拟对话 — 看每次发送了什么

0 tokens
点击添加消息，看数据怎么增长
这意味着什么
对话越长，每次请求越贵。对话到第100轮，你的第101条消息会把前100条全部打包重发一遍。这就是上下文窗口（Context Window）——它有上限，超出就"失忆"，旧消息会被截断。

免费版通常上下文更短，Pro版更长，Enterprise版可以更长——这不是故意限制，是成本所致。

上下文窗口的大小也决定了AI能处理多长的文档。Claude的上下文窗口目前高达20万Token——大约是一整本小说。这需要消耗大量算力。

05
System Prompt

你看不到的第一条消息

每次你打开claude.ai、ChatGPT或任何AI产品开始对话，在你输入第一个字之前，系统已经悄悄往对话里塞进了一段指令——这就是 System Prompt（系统提示词）。

它告诉AI：你是谁、你能做什么、你不能做什么、你应该用什么语气——所有产品层面的"个性"，都来自这里。

System: 你是Claude，由Anthropic创建的AI助手。你应该诚实、无害、有帮助。你有搜索网络、执行代码、分析文件等工具。用户正在使用claude.ai，界面支持Markdown渲染……

（实际长达数千字，你永远看不到完整版）
System: 你是"小慧"，XX公司的客服助手。只回答关于我们产品的问题。不要讨论竞争对手。用轻松友好的语气……

同一个Claude模型，装进不同System Prompt，就变成了完全不同的产品
（没有System Prompt）

你直接接触到了原始模型。没有产品包装，没有限制，没有工具。Claude会展现训练时的"默认行为"，而不是任何产品的定制行为。
为什么同一个模型在不同平台表现不同
System Prompt决定了一切"性格"细节：语气、限制、工具、记忆方式。这也是为什么你可以用API做出和claude.ai完全不同感觉的产品——你只是换了一个System Prompt。
06
产品层

claude.ai 在裸API之上做了什么

claude.ai不是"裸模型"。它是Anthropic自己用Claude API构建的产品——和你用API构建产品，是同一个原理。

它在裸模型上叠加了很多层：

产品包装层（你感知不到的部分）
① 系统提示词 — 几千字的指令，定义Claude的行为边界

② 对话历史管理 — 自动打包历史消息一起发送，超长时智能压缩

③ 工具接入 — 联网搜索、代码执行、文件解析、图像生成——这些不是模型自带的，是产品层接入的

④ Artifacts渲染 — 你看到的代码块、可运行的网页——产品层识别并渲染的

⑤ Projects / Memory — 跨对话"记忆"，本质是把记录注入System Prompt

⑥ 用户认证 / 限流 — 知道你是谁，按套餐控制用量

所以"claude.ai的Claude"和"原始API的Claude"是同一个模型，但体验完全不同——因为包装层不同。这也意味着：任何人都可以用API构建一个类似claude.ai的产品，只是少了那些精心设计的包装。

算不算Agent？
claude.ai有部分Agent特征——它能调工具、管理上下文、执行多步操作。但主循环仍然是"你发、它回"，不会自主持续执行。Claude Code才是真正的Agent——它会自己规划、执行、纠错、循环，直到任务完成，不需要你每步确认。

09
工具调用

AI 如何连接真实世界

原始LLM只能接收文字、输出文字——它不能搜索网页、不能查天气、不能访问数据库、不能执行代码。它被困在训练数据里。

Tools（工具调用 / Function Calling）打破了这个限制。

工作原理
你告诉AI："你有一个工具叫search_web，需要的时候你可以调它。"

当AI判断需要搜索时，它不是直接搜索——它输出一个结构化的"调用请求"，你的程序读到这个请求、真正去搜索、把结果返回给AI、AI再继续回答。

AI本体从未离开"文字进出"的模式——是你的程序在帮它执行真实动作。
// AI决定要搜索时，它的"输出"长这样： { "type": "tool_use", "name": "search_web", "input": { "query": "今日上证指数" } } // 你的程序收到这个，去真正搜索，再把结果发回： { "role": "tool", "content": "上证指数今日收于3124.5点，涨0.8%" } // AI拿到结果，继续生成回答

claude.ai的联网搜索、代码执行、文件分析——都是这个原理。只不过Anthropic帮你实现了这些工具，并接入了产品里。你用API也可以自己接入任何工具——这就是定制化AI的基础。

可以接入哪些工具
任何可以用代码调用的东西：搜索引擎、数据库、发邮件、发Slack消息、查股价、控制智能家居、调用另一个API……工具边界等于你代码能做到什么的边界。

Agent 智能体

AI 从回答问题
到完成任务

普通AI：你问，它答。一来一回。你要驱动每一步。

Agent：你给一个目标，它自己制定计划、调用工具、执行步骤、根据结果调整、循环直到完成——你只需要等结果。

这是质变，不是量变。模拟一个真实Agent的执行过程：

任务：帮我研究竞争对手A公司，总结他们最新的产品动态
Agent的核心：循环
Agent不是"调用一次AI"，而是一个循环：

规划 → 行动 → 观察结果 → 重新规划 → 再行动 → ……

每一轮循环都是一次完整的API调用。一个复杂任务可能循环几十次。这就是为什么Agent比普通聊天贵得多——它消耗的Token是普通对话的几十倍。

Claude Code是目前最典型的Agent产品——你给一个编程任务，它自己读代码、写代码、执行测试、修复错误、循环直到任务完成。你可以去做别的事，回来看结果。
